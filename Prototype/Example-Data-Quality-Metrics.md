# Example Data Quality Metrics

## Purpose

One objective of the Kinshasa pilot is to evaluate whether routine clinical information can be captured in a structured and reusable format.

The following metrics illustrate how data quality may be assessed.

## Completeness

Definition:
Percentage of required fields containing valid values.

Example:

| Variable      | Completeness |
| ------------- | ------------ |
| Sex           | 100%         |
| Disease Group | 100%         |
| Hemoglobin    | 80.5%        |
| BMI           | 66.13%       |

## Consistency

Definition:
Logical agreement between related variables.

Examples:

* BMI corresponds to recorded height and weight.
* Mortality status is recorded only once per patient.
* Treatment category matches disease group.

## Timeliness

Definition:
Delay between clinical encounter and structured data entry.

Example target:

* Median delay < 7 days

## Standardization

Definition:
Use of predefined categories and controlled vocabulary.

Examples:

* Disease group restricted to predefined categories.
* Consistent facility identifiers.
* Standardized treatment classifications.

## Pilot Success Indicators

Illustrative targets:

* Completeness > 85%
* Consistency > 95%
* Timeliness < 7 days
* Standardization > 95%
